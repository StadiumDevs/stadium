---
description: Read a GitHub issue and produce a scoped implementation plan as a PR-ready comment. No code.
argument-hint: <issue-number>
---

Produce an implementation plan for GitHub issue **#$ARGUMENTS** and post it as a comment on the issue. Do not write code.

## Steps

1. `gh issue view $ARGUMENTS --json number,title,body,labels` — read it.
2. Use the `stadium-explorer` subagent (thoroughness: medium) to map affected files.
3. Draft a plan with these sections:
   - **Understanding** — restate goal and acceptance criteria in your own words
   - **Files to change** — paths with short reason each
   - **New files** — paths and purpose
   - **Utilities to reuse** — paths
   - **Tests** — what to add, where
   - **Invariants at risk** — what you'll be careful about
   - **Out of scope** — what you will not touch
   - **Estimated complexity** — trivial / small / medium / large
   - **Open questions** — for the issue author / user
4. Post the plan as an issue comment:
   ```bash
   gh issue comment $ARGUMENTS --body-file <(cat <<'EOF'
   ## Implementation plan (draft — not approved yet)

   <plan body>

   ---
   *Generated by `/triage-issue`. Implementation requires explicit approval via `/ship-issue`.*
   EOF
   )
   ```
5. Report to the user: issue URL and a one-sentence summary of the plan.

**Never implement from this command.** This command ends at "plan posted."
